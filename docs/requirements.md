# Requirements

## Functional Requirements

RF-01: The system must allow users to register and authenticate.
RF-02: The system must allow users to view available resources.
RF-03: The system must allow users to create and cancel bookings.
RF-04: The system must prevent overlapping bookings for the same resource and time.
RF-05: The system must allow administrators to manage resources and schedules.

## Non-Functional Requirements

RNF-01: The API must use token-based authentication (JWT).
RNF-02: The system must follow a layered architecture.
RNF-03: The API must expose consistent error responses.
RNF-04: The system must be testable through automated tests.
RNF-05: The API must be documented using OpenAPI / Swagger.
